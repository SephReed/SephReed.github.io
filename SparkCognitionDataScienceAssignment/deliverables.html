Q & A

Describe your model and why did you choose this model over other types of models?
The model I chose in this case was whatever the quickest simplest thing I could do from scratch was: a series of 1 to 1 correlations between the result and every other field, weighted where the divergence is strongest. This model is not ideal as it only correlates things in a 1:1 way, unlike deep learning with a stronger node depth. What is interesting about the model I used is that is produces something I may have made up called a "rule set" which awards percentages of likelihood to one option or the other based off the value in the field. I think this is a pretty cool thing because it can easily be saved to and loaded from string, and can even be calculated by hand. The percentages don't add up to 100% for each datum, but instead 100% is divided up amongst every reward.

Describe any other models you have tried and why do you think this model preforms better?
I have tried no others. Big data analysis is not something that was taught in my school, and I've never had it as a job. I have done some superficial studying of deep learning algorithms, but have not yet implemented one.

How did you handle missing data?
If data was missing, the size of the dataset was decremented.

How did you handle categorical (string) data? How did you handle unbalanced data?
Categorical data was grouped. Unbalanced data (numerical?) was divided into localized chunks (based off 10x the average step size between values) and the normal distribution curve was computed for each chunk. I did not have time to figure out how to compare all the different chunks, and so opted to simply compare the most common curve for each outcome.
