Robot Soul - (Single Page Web App, Digital Audio Workstation, Js Audio Node)

Robot Soul is an in browser DAW (Digital Audio Workstation) personal project of mine. It has movable devices with front and back ends. The front end contains the knobs and general controls, while the back shows ports for input and output which can be connected to other devices. Projects can be saved and loaded, and music can be played using the keyboard. It's still in it's early stages, but I hope to someday use this as a high functioning performance tool. Lullaby Sound Interactive Machine - (Python, Raspberry Pi, Interactive Art, Music Theory, Audio Editing)

The Lullaby Interactive Sound Machine was the auditory aspect on the center piece of the Flipside Art Festival for 2016. The fingernails of a massive wooden baby were all pressable, and triggered constantly progressing musical notes. The notes were re-chosen every five seconds to be a new chord, or occasionally a new key, which gave the piece a beautifully progressive yet chaotic sound as participants played around with little regard for melody or rhythm. The buttons also triggered flame throwers atop the Baby, occasionally starting a pattern which was generated by the Effigy Flame Effects Sequencer (project showcased further below). View Source Code

PINE - (User Friendly Framework and Documentation, Asynchronous Data Structures, Html DOM)

PINE is a UI framework I made in order to streamline the process of making special UI elements, especially ones which communicate or depend on the actions of others, without having issues with excessive event or mutation listeners. It has become the only tool I use, replacing JQuery and AngularJs, and I really enjoy it. That being said, I'm one person alone working on it, and I can not call this a business ready solution, but for personal projects it is amazing. To explain it simply, it grows outwards from a set root point in stages, effecting page elements which match specified cases, and then it modifies those elements sometimes adding functions. Each stage either depends on the completion of previous stages in parent elements or child elements, where some types of functionality grow from the root and others gather from the sub elements. If you'd like to know more, please ask. Proper GUI growing is a subject I'm very fond of. View Source Code

Black Lister - (Web Extensions)

I got really tired of w3schools and about.com coming up in my searches so I made a Firefox extension to "Blam" them. It's a pretty straightforward extension, if you find a page you don't like, you open the extension and click the big red button to "Blam" it. From then on, any external links to the page will be blurred out and desaturated. In the other direction, if you really like a page, you can highlight it. It's a very good tool for search engines. Code Ethics

Open Source

Too many great ideas have been thwarted by the closed source systems which inevitably fall into the wrong hands. I don't want to spend my life creating ideas for a thing that I may some day be denied access from, nor do I wish to deny the world of the tools I create.

No Bloat

It's just a matter of cleanliness. There are a lot of users and businesses that will slap together a few oversized libraries and consider it "working." I prefer to use code as modules where you can easily mix, match, and replace the pieces and deliver a product that is efficient at doing well defined tasks.

Explorative

In order to keep myself and the things I make relevant, I always aim to focus on the future. Often this means trying new things and being on the front of the technology adoption bell curve.

Humane

The world is as hurt as it ever needed to be and some. If I have to choose between making the world a worse place and making my life a little harder, I'll take the hard route.

My Toolset

HTML 5, CSS 3, Javascript

I've been working with HTML since age eleven, CSS the past six years, and Javascript the past four years. Javascript is one of my newest languages but it is also one of my favorites, mostly due to its portability and transparency.

PINE

PINE is a tool I built myself for creating functional user interfaces. At this point it has completely replaced JQuery and Angular Js in my personal projects, with astonishing results.

Zepto (JQuery, JQuip)

Zepto, JQuery, and JQuip are all plugins which help with dom traversal using the standard $('selecter') syntax. The only differences are that JQuery is really bloated, JQuip is the modular version of JQuery, and Zepto is built from scratch for the sake of being thin. If you know one, you know all three.

Green Sock Animations Plugin

GSAP is a really powerful web animations plugin. It's not Flash (thank heavens), but it can do a lot of flashy animations using some fancy css manipulations and a good bit of hardware acceleration.

Cordova

Cordova is the open source tool that Adobe copied, changed for the worse, and then pasted their name on as "PhoneGap." It's a bridge between web design and phone apps (cameras, microphones, contact lists, ect.) that makes it easy to develop for all devices at the same time.

MySQL

MySQL is the database manager I'm most familiar with, but they're all fairly similar. I've built a couple websites around big data dreams, and took an entire class once in SQL jedi-ism.

Three.js

Currently, this is my favorite new tool. It's what I used to make the moving 3d logo at the top of the page and the "3d Walkscape" showcased below. In my opinion, Three.js is, without any doubt, the future of the web. It's incredible.

Java

Java was my bread and butter for a long time, but I've since moved to Javascript for the sake of portability and easier UI control.

C++, Python, GoLang, AngularJs, PHP

These are all languages and libraries I've worked with, but were not productive enough for me to create anything truly presentable. I don't use them anymore, but I still know them fairly well.

Unix/Linux

I work in OSX, but I've dabbled with a few Linux distrubutions as well. Namely Ubuntu, Mint, and Raspian. I liked them all and would have been content to call one of them my new OS (Mint), but there were a few essential things that I couldn't get running the way I like, and so I came back to OSX. Some day, though, I'll have weened myself off of closed source companies for good.
