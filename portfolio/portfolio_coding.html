Code Ethics

Open Source

I just flat out don't like beurocracy, so working on open source software is my way to share and hone my skills without having to feel compromised. Besides, there is nothing in the world that's better at getting between a good idea and its implementation than closed source code and a "busy" manager.

No Bloat

It's just a matter of cleanliness. There are a lot of users and businesses that either refuse to use or don't understand updates and working around them often creates bloat. These aren't the kind of people nor habits I care to burden myself with.

Explorative

It's not exactly an ethic, but I'm much more enthused about the future than I am the past. Certainly, many great things have been done, but rather than learning to recreate or emulate them, I prefer to get my hands dirty trying to explore the unexplored. Pioneering, in a modern sense.

Humane

The world is as boned as it ever needed to be and some. But more importantly, the best things in my life could never have been paid for, they came from a certain type of chaos. It's a strange mixture of goodwill, resources, and freedom that creates the greatest stories, and I really like a good story.

My Toolset

HTML 5, CSS 3, Javascript

I've been working with HTML since age 12, CSS the past 6 years, and Javascript the past four years. Javascript is one of my newest languages, but it is also one of my favorites, mostly due to its portability and transparency.

Zepto (JQuery, JQuip)

Zepto, JQuery, and JQuip are all plugins which help with dom traversal using the standard syntax. The only differences are that JQuery is really bloated, JQuip is the modular version of JQuery, and Zepto is built from scratch for the sake of being thin. If you know one, you know all three.

Green Sock Animations Plugin

GSAP is a really powerful web animations plugin. It's not Flash (thank heavens), but it can do a lot of flashy animations using some fancy css manipulations and a goot bit of hardware acceleration.

Cordova

Cordova is the open source tool that Adobe copied, changed for the worse, and then pasted their name on as "PhoneGap." It's a bridge between web design and phone apps (cameras, microphones, contact lists, ect.)

MySQL

MySQL is the database manager I'm most familiar with, but they're all fairly similar. I've built a couple websites around big data dreams, and took a class in SQL jedi-ism.

PINE

PINE is a tool I built myself for creating functional user interfaces. It's pretty rad, and there's more about it in my "Showcase" section.

Three.js

Currently, this is my favorite new tool. It's what I used to make the moving 3d logo at the top of the page and the "3d Walkscape" showcased below. In my opinion, it is without any doubt the future of the web. It's incredible.

Java

Java was my bread and butter for a long time, but I've since moved to Javascript for the sake of portability and UI tools (which html/css have plenty of).

C++, Python, GoLang, PHP

These are all languages I've worked with, but were not productive enough for me to create anything truly presentable. I don't use them anymore, but I still know them fairly well.

PINE - (User Friendly Framework and Documentation, Asynchronous Data Structures, Html DOM)

This is a deep one. The amount of things which went into this framework are pretty expansive. I can't really do it justice in so few words, but I can say I'm very proud of it. 3d Walkscape - (3d, Game Dev)

(current) For this project I'm using Three.js, SketchUp, and Blender to make a small, scenic, 3d world. So far it has realistic looking water, perlin noise generated hills, a beautiful skybox, randomly placed trees, some very rough physics emulation, and a little house with a door you can open and close. The project is only barely started, but the hope is that it will someday be the beautiful pine forest land I will probably never own.
Check it out!

Effigy Flame Effects Sequencer - (Java)

What started off as a simple way to test what a group of flame throwers would look like when triggered by buttons eventually evolved into an entire sequencing program which can import and export MIDI, and has its own propriety file extension for projects called a .feg (F E G == Effigy). The program also has algorithms which can be combined with different patterns to create even crazier patterns. This year it will be premiering at a small festival called "Flipside," and the hopes for it are high.

Black Lister - (Web Extensions)

I got really tired of w3 schools coming up in my searches so I made a chrome extension to "Blam" it. Unfortunately, upon uploading the finished extension, I got an email from Google saying that my developer account was suspended and all my extesions were taken down without any explanation as to why and no extra information I might use to fix the situation. Eventually (a week later) my account was re-instated, also without any explanation, but by that point I'd already sworn off using Google products as much as possible. It's been a lot like dieting. At first is was kind of hard, but, looking back, Google is kind of icky. Anyways, the point is that I abandoned this project as a result, but will eventually be porting it to Firefox.
If you use Chrome, check it out!

Banking App - (Rapid Prototyping)

My girlfriend is in school for interaction design and had to redesign a banking app. Unfortunately, neither her nor most people in the field of design know how to use HTML/CSS, so instead they use Illustrator, which is cofoundingly unresouceful. It got to the point where the time and stress of the project were getting hard for me to handle, so I redesigned her entire app as a working prototype, pixel for pixel, in about a day. Then I proceeded to fork the project and redesign it the way I think a banking app should be. Had I been doing this in illustrator, I would have had to start from scratch, which is fucking crazy! But in the end, I realized the ability to think of something and then make it takes a lot longer to learn in html/css than it does in Illustrator, so I guess it makes sense that people use that tool instead.

I never ended up porting the webapp to an actual phone but I could do it almost effortlessly using Apache Cordova, or Ionic if need be.
