Robot Soul - (Single Page Web App, Digital Audio Workstation, Js Audio Node)

Robot Soul is an in browser DAW (Digital Audio Workstation) personal project of mine. It has movable devices with front and back ends. The front end contains the knobs and general controls, while the back shows ports for input and output which can be connected to other devices. Projects can be saved and loaded, and music can be played using the keyboard. It's still in it's early stages, but I hope to someday use this as a high functioning performance tool.

This project is currently at a stand still, and the live demo is down. I will be starting this project again as soon as Audio Worklets are implemented in Firefox and Chrome (2018 at best). View Source Code

Lullaby Sound Interactive Machine - (Python, Raspberry Pi, Interactive Art, Music Theory, Audio Editing)

The Lullaby Interactive Sound Machine was the auditory aspect on the center piece of the Flipside Art Festival for 2016. The fingernails of a massive wooden baby were all pressable, and triggered constantly progressing musical notes. The notes were re-chosen every five seconds to be a new chord, or occasionally a new key, which gave the piece a beautifully progressive yet chaotic sound as participants played around with little regard for melody or rhythm. The buttons also triggered flame throwers atop the Baby, occasionally starting a pattern which was generated by the Effigy Flame Effects Sequencer (project showcased further below). View Source Code

PINE - (User Friendly Framework and Documentation, Asynchronous Data Structures, Html DOM)

PINE is a UI framework I made in order to streamline the process of making special UI elements, especially ones which communicate or depend on the actions of others, without having issues with excessive event or mutation listeners. It has become the only tool I use, replacing JQuery and AngularJs, and I really enjoy it. That being said, I'm one person alone working on it, and I can not call this a business ready solution, but for personal projects it is amazing. To explain it simply, it grows outwards from a set root point in stages, effecting page elements which match specified cases, and then it modifies those elements sometimes adding functions. Each stage either depends on the completion of previous stages in parent elements or child elements, where some types of functionality grow from the root and others gather from the sub elements. If you'd like to know more, please ask. Proper GUI growing is a subject I'm very fond of. View Source Code

Black Lister - (Web Extensions)

I got really tired of w3schools coming up in my searches above MDN so I made a Firefox extension to "Blam" it. It's a pretty straightforward extension, if you find a page you don't like, you open the extension and click the big red button to "Blam" it. From then on, any external links to the page will be blurred out and desaturated. In the other direction, if you really like a page, you can highlight it. It's a very good tool for search engines. Code Ethics

Open Source

I love open source so much. If at all possible, I really want to be part of the movement. People paving the way for others to pave the way for others is a noble idea, and I believe in it.

No Bloat

I like to do all of my projects from a blank file, using vanilla code. The result is that whatever code is there, is the code I need, and no more. Also, I learn a lot. Obviously not all projects can (or should) be like this, but I still always do my best to find and pick whichever code base is most modular, and most to the point.

Exploratory

In order to keep myself and the things I make relevant, I always aim to focus on the future. Often this means trying new things and being on the front of the technology adoption bell curve. Obviously, backwards compatibility is crucial, but similarly, something that worked in the past might not really "work" today.

Humane

The world is as hurt as it ever needed to be and some. There aren't many things I'm unwilling to work with, but cruelty for profit one of them. Moreover, if there is something I can be doing to make the world a better place, I will be far more self motivated than otherwise.

My Toolset

I've been working with HTML since age eleven, CSS the past six years, and Javascript the past four years. Of all of these, Javascript is by far my favorite, and with the advent of Node.js, I rarely find a problem that can't be fixed with it.

PINE

PINE is a tool I built myself for creating functional user interfaces. At this point it has completely replaced JQuery and Angular Js in my personal projects, with astonishing results.

MySQL

MySQL is the database manager I'm most familiar with, but they're all fairly similar, and I have no allegiance to any one of them. I've built a couple websites around big data dreams, and taken classes SQL jedi-ism.

Three.js

3d animation is not particularly common on the web these days, but THREE.js is showing a lot of potential. It's pretty cool stuff, and I really enjoy playing with it.

Java

Java was my bread and butter for a long time, but I've since moved to Javascript for the sake of portability and easier UI control.

C++, Python, GoLang, AngularJs, PHP

These are all languages and libraries which fall somewhere in between me having learned them, and me having chose them for a long term project. C++ has gotten significantly more of my time than the others having been the code my college chose for teaching, but I still only ever used it for many short term assignments.

I liked them all and would have been content to call one of them my new OS (Mint), but there were a few essential things that I couldn't get running the way I like (multi-touch gestures), and so I came back to OSX.
