Overview

What PINE is:
PINE is a HTML triggered functionality injector. It allows the user to create and use special tags and attributes to make HTML objects more functional and also more indicative of what their function is.

Why one might use it:
The best reason to use PINE is if you like its syntax. PINE may some day become what JQuery now is (depreciated) but the syntax that JQuery introduced is still ubiquitous (see Zepto, JQuip, or Stack Overflow). Another good reason is that PINE is dedicated to being open source, having no bloat, remaining open to new things, and treating people humanely. Actually, it's just me. I'm dedicated to those things (~Seph).

Why one might not:
The best reason to not use PINE is if you're trying to support old or generally disfunctional browsers. Another good reason would be if you need a lot of help with programming. The code is pretty easy to peruse, and there will be comments to read, and these docs as well, but it's not popular enough to have a support community that you can depend on.

Why not to use AngularJS instead:
AngularJS is what inspired this, but I abandoned it due to its stanky case of "feature creep." It is still a great choice, though, for those who want something that is packaged with tons of features (all hail "ng"), and is designed to work with the average GM and republican grandma in their campaign to not use the latest version of Firefox.

Why not to use React instead:
React was made to be very fast and it is. But it is also an eyesore of fragile and unimportant information when it comes to desigining a GUI. To elaborate, when designing a GUI, I think the things you see in your periphera should be indicative of what the user will see, rather than the completed functions you could have tucked away a long time ago. Also, one should feel comfortable typing when working on a GUI, but being a few lines away from creating a compiling error tends to have the opposite effect on most people. HTML in Javascript remains a bad idea.
